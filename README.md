MINIGUIA DE ESTUDO
1. Resumos Estruturados do Assunto
I. Fundamentos e o Papel do Designer de UX
O UX Design foca na interação que os usuários têm com produtos (sites, aplicativos e objetos físicos), buscando torná-los úteis, agradáveis e acessíveis. O papel de um designer iniciante envolve empatia, pesquisa, criação de wireframes, mockups e protótipos. O mercado valoriza a capacidade de basear o design em fatos e dados, em vez de suposições ou "adivinhação".
II. Framework: Design Thinking
É tanto uma ideologia centrada no usuário quanto um processo de seis etapas para resolver problemas:
Empatia: Coleta de dados qualitativos para entender necessidades reais.
Definição: Identificar o problema central e as necessidades a serem atendidas.
Ideação: Gerar uma ampla gama de soluções potenciais.
Prototipagem: Manifestar ideias em formatos testáveis.
Teste: Validar se a solução atende às necessidades e é usável.
Implementação: Colocar a visão em prática para impactar a vida do usuário.
III. Pesquisa com Usuários (User Research)
A pesquisa é a investigação sistemática para informar o design. Ela se divide em:
Qualitativa: Foca em como as pessoas pensam e sentem (opiniões, motivações) através de entrevistas e estudos de campo.
Quantitativa: Gera dados numéricos e estatísticos por meio de pesquisas estruturadas e análises de cliques.
Importância: Evita desperdício de tempo e dinheiro ao garantir que o produto resolve problemas reais e possui vantagem competitiva.
IV. Design Sprints
Um processo de cinco dias para validar ideias e resolver grandes desafios:
Segunda (Entender): Mapear o problema e escolher o foco.
Terça (Idear): Esboçar soluções (usando métodos como o Crazy 8s).
Quarta (Decidir): Escolher a melhor solução para prototipar.
Quinta (Prototipar): Construir um protótipo fiel o suficiente para parecer real.
Sexta (Testar): Coletar feedback de 5 usuários reais para identificar 85% dos problemas de usabilidade.
V. Acessibilidade Global e Design Inclusivo
Projetar para todos significa considerar deficiências físicas e cognitivas, além de fatores ambientais (como sol forte ou telas quebradas).
Estratégias: Usar linguagem simples (evitar jargões), combinar texto com ícones para diferentes níveis de alfabetização e respeitar a privacidade de quem usa tecnologias assistivas.
Representação: Incluir diversidade (raça, habilidades, classes sociais) em ilustrações e personas para evitar exclusão por "suposições de usuário médio".

2. Glossário de Conceitos Principais
Acessibilidade (Accessibility): O design de produtos, dispositivos ou ambientes para pessoas com deficiências.
Crazy 8s: Exercício de ideação onde se esboçam oito variações de uma solução em oito minutos.
Design Centrado no Usuário: Abordagem que coloca as necessidades dos usuários no centro de cada decisão de design.
Design Sprint: Framework de cinco dias criado pelo Google Ventures para acelerar a resolução de problemas.
Empatia (Empathy): A capacidade de entender e compartilhar os sentimentos e necessidades de outra pessoa sem preconceitos.
How Might We (Como poderíamos): Método para transformar problemas em oportunidades de solução.
Interseccionalidade: Reconhecimento de que gênero, religião ou classe podem influenciar a forma como pessoas com deficiência acessam a tecnologia.
Protótipo (Prototype): Uma representação testável de um produto, que pode ser de baixa ou alta fidelidade.
Wireframe: Um esboço básico da estrutura de uma interface, focando na hierarquia e funcionalidade.

3. Prompts Reutilizáveis para Revisão
Para Simular Cenários de Pesquisa:
"Atue como um usuário com baixa alfabetização digital. Analise [insira uma funcionalidade de app] e descreva quais barreiras de acessibilidade você encontraria com base nos princípios de Acessibilidade Global do Google."


Para Refinar Ideação:
"Tenho um problema de design: [descreva o problema]. Aplique a técnica 'How Might We' para gerar 5 oportunidades de solução focadas em design inclusivo."


Para Planejar Sprints:
"Estruture um cronograma de Design Sprint de 5 dias para resolver [descreva o desafio], definindo quem seriam os papéis ideais (Decisor, Facilitador, etc.) e as atividades principais de cada dia."


Para Crítica de Design:
"Revise este fluxo de interface: [descreva o fluxo]. Identifique possíveis falhas de usabilidade usando a perspectiva de pesquisa qualitativa e sugira como validar isso com usuários reais."

4. Flashcards para o Anki
Card 1
Frente: Qual a diferença fundamental entre Pesquisa Qualitativa e Quantitativa em UX?
Verso: A Qualitativa foca no "porquê" (pensamentos, sentimentos e motivações), enquanto a Quantitativa foca no "quanto" (dados numéricos, estatísticas e padrões de cliques).
Card 2
Frente: Quais são as 6 etapas do processo de Design Thinking?
Verso: 1. Empatia; 2. Definição; 3. Ideação; 4. Prototipagem; 5. Teste; 6. Implementação.
Card 3
Frente: Por que testar com apenas 5 usuários em uma Design Sprint é considerado suficiente?
Verso: Segundo o modelo de Nielsen, entrevistar 5 usuários permite identificar aproximadamente 85% dos problemas de usabilidade do produto.
Card 4
Frente: O que significa o conceito de "Goldilocks quality" em prototipagem?
Verso: Criar um protótipo com qualidade "na medida certa": real o suficiente para que o usuário acredite nele, mas sem gastar tempo excessivo em perfeccionismo.
Card 5
Frente: Cite três estratégias para melhorar a acessibilidade global em aplicativos.
Verso: 1. Usar inglês simples e evitar jargões; 2. Combinar texto com ícones (dicas gráficas); 3. Evitar menus com hierarquias complexas para facilitar leitores de tela.
